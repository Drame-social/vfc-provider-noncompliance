## Stronger post

I completed a public health analytics portfolio project that mirrors a CDC-style provider compliance monitoring workflow.

Project: Synthetic VFC Provider Non-Compliance Analytics

What I built:
- Synthetic provider and site-visit datasets
- Complete data dictionary and validation rules
- Python pipeline to generate, validate, and analyze the data
- SQL scripts for schema creation, QA checks, and analytic queries
- SAS script for frequency tables, chi-square testing, and logistic regression
- Power BI DAX measures for dashboard development
- GitHub-ready documentation: README, methods, limitations, results summary, and QC checklist

Public health question:
How does provider non-compliance vary over time and across program-relevant characteristics such as region, provider type, provider size, urbanicity, visit method, and time in program?

This project reflects my background in public health analytics, surveillance data quality, immunization/VFC-style compliance analysis, SAS, SQL, Python, Power BI, and program reporting.

All data are synthetic. No real CDC, provider, patient, or restricted site-visit data are included.
